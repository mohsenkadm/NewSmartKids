# تعديلات المشروع - نظام رفع الصور والفيديو للمنتجات

## نظرة عامة
تم إجراء تعديلات على المشروع لدعم رفع الصور والفيديو مع عملية إضافة/تعديل المنتجات مباشرة، بدلاً من رفعها في عملية منفصلة.

## التعديلات الرئيسية

### 1. تعديل نموذج البيانات (Entity)
**الملف:** `WebSmartKid\Models\Entity\Images.cs`
- إضافة حقل `IsVideo` (bool): لتمييز ما إذا كان العنصر صورة أو فيديو
- إضافة حقل `VideoGuid` (string): لحفظ معرف الفيديو من Bunny Stream

### 2. تعديل قاعدة البيانات
**الملف:** `DatabaseScripts\UpdateImagesTable.sql`
تم إنشاء سكريبت SQL لإضافة الأعمدة الجديدة:
```sql
ALTER TABLE [dbo].[Images] ADD [IsVideo] BIT NOT NULL DEFAULT 0
ALTER TABLE [dbo].[Images] ADD [VideoGuid] NVARCHAR(255) NULL
```

**خطوات التنفيذ:**
1. افتح SQL Server Management Studio
2. اتصل بقاعدة البيانات الخاصة بك
3. نفذ السكريبت الموجود في `DatabaseScripts\UpdateImagesTable.sql`

### 3. تعديل Controller
**الملف:** `WebSmartKid\Controllers\ProductsController.cs`

**التعديلات:**
- إضافة `IStorageServices` في Constructor
- إضافة Method جديد `PostWithFiles` يدعم:
  - حفظ المنتج
  - رفع صور متعددة باستخدام `UploadService`
  - رفع فيديو واحد على Bunny Stream
  - حفظ URLs الخاصة بالصور والفيديو في جدول Images

**المزايا:**
- الصور والفيديو يتم رفعها مع عملية الحفظ في طلب واحد
- استخدام `IStorageServices` للرفع بشكل منظم
- الفيديو يتم رفعه على Bunny Stream تلقائياً
- يتم حفظ URL الفيديو في جدول Images مع تمييزه كفيديو

### 4. تعديل واجهة المستخدم
**الملف:** `WebSmartKid\Views\Home\Products.cshtml`

**التعديلات:**
- إضافة حقل رفع الفيديو (input type="file" للفيديو)
- تحديد أن الفيديو يجب أن يكون واحد فقط
- إضافة accept="video/*" لحقل الفيديو
- إضافة accept="image/*" لحقل الصور

### 5. إعادة تنظيم ملف JavaScript
**الملف:** `WebSmartKid\wwwroot\js\Products.js`

**التحسينات:**
- تنظيم الكود بشكل أفضل مع تقسيمه إلى أقسام:
  - متغيرات عامة
  - دوال عرض البيانات
  - دوال العمليات (CRUD)
  - دوال الأعمار
  - دوال التحديث والعرض
- إضافة تعليقات JSDoc لكل دالة
- إضافة دالة `saveProductWithFiles()` لرفع المنتج مع الصور والفيديو
- إضافة دالة `clearProductForm()` لمسح النموذج
- تحديث `filltableImage()` لعرض الفيديو باستخدام video tag

**الدوال الجديدة/المحدثة:**
- `saveProductWithFiles()`: حفظ المنتج مع الصور والفيديو في طلب واحد
- `clearProductForm()`: مسح جميع حقول النموذج
- `filltableImage()`: عرض الصور والفيديوهات مع التمييز بينهما

### 6. تعديل Entity Configuration
**الملف:** `WebSmartKid\Models\EntityMap\ImagesMap.cs`
- إضافة تكوين للحقول الجديدة `IsVideo` و `VideoGuid`

## آلية العمل الجديدة

### إضافة منتج جديد:
1. المستخدم يملأ بيانات المنتج
2. المستخدم يختار صور متعددة (اختياري)
3. المستخدم يختار فيديو واحد (اختياري)
4. عند الضغط على "حفظ":
   - يتم حفظ بيانات المنتج أولاً
   - يتم رفع الصور إلى السيرفر المحلي باستخدام `UploadService`
   - يتم رفع الفيديو إلى Bunny Stream
   - يتم حفظ URLs الصور والفيديو في جدول Images
   - الفيديو يُحفظ مع `IsVideo = true` و `VideoGuid` يحتوي على معرف الفيديو

### عرض الصور والفيديوهات:
- الصور تعرض كـ `<img>` tags
- الفيديوهات تعرض كـ `<video>` tags مع دعم HLS streaming

## ملاحظات مهمة

1. **قاعدة البيانات:**
   - يجب تنفيذ السكريبت SQL قبل استخدام الميزات الجديدة
   - السكريبت يتحقق من وجود الأعمدة قبل إضافتها

2. **Bunny Stream:**
   - تأكد من أن `BunnyClient` مُعرف بشكل صحيح في `Startup.cs` أو `Program.cs`
   - تأكد من صحة مفاتيح API الخاصة بـ Bunny Stream

3. **حدود الفيديو:**
   - كل منتج يمكنه الحصول على فيديو واحد فقط
   - يمكن للمنتج الحصول على صور متعددة

4. **التوافق:**
   - الكود القديم للرفع المنفصل لا يزال موجوداً في `UploadFile3Async` للتوافق
   - يُنصح باستخدام الطريقة الجديدة `PostWithFiles`

## اختبار التعديلات

1. نفذ SQL Script
2. قم ببناء المشروع
3. جرّب إضافة منتج جديد مع صور وفيديو
4. تحقق من رفع الصور والفيديو بنجاح
5. تحقق من عرض الصور والفيديوهات في واجهة "عرض الصور"

## المشاكل المحتملة وحلولها

### المشكلة: الفيديو لا يتم رفعه
**الحل:** تحقق من:
- إعدادات Bunny Stream API
- حجم الملف (قد يكون هناك حد أقصى)
- صيغة الفيديو المدعومة

### المشكلة: الصور لا تظهر
**الحل:** تحقق من:
- مسار حفظ الصور في السيرفر
- صلاحيات الكتابة على المجلد
- الرابط الأساسي `Key.CurrentUrl`

### المشكلة: خطأ في قاعدة البيانات
**الحل:** 
- تأكد من تنفيذ SQL Script
- تحقق من connection string
